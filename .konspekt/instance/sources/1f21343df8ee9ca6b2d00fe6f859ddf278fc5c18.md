## Human — the three differentiators
the difference between konspekt and agent-kept memory is that konspekt is
a) human-curated (you decided what goes in and when) - you don't run into risk
of LLM dropping or ignoring stuff
b) has provenance (each decision/note/fact is linked to actual conversation text)
c) has a public data model/structure (interop)
the users are primarily humans working on projects, but nothing stops one from
feeding .konspect/instance stuff to LLM as RAG
Claude's memory is an overview/summary of a project.
konspekt is what it says - all notes, written verbatim and neatly organized.

## Assistant — machine-authored vs human-authored; who the user is
"Mem0, Zep, and platform memory (Claude Projects / ChatGPT / Gemini Gems) are
all machine-authored, machine-read: a model decides what's salient, extracts a
compressed version, and stores it to feed back to a model. The reader is the
agent. konspekt inverts both terms — human-authored, human-read, incidentally
machine-readable. That single line answers 'who is the user?': the user is the
human; agent memory's user is the agent."

## Assistant — integrity, not completeness (the claim not to overclaim)
"What it guarantees is integrity, not completeness: nothing enters the record
that you didn't approve, and what's there is verbatim. That's the defensible
claim. Frame it as 'you control what's in the record,' not 'the LLM never misses
anything.'"

## Assistant — the layers compose, not collide
"A konspekt instance is a clean, human-validated source you can feed into
Mem0/Zep/RAG. It's upstream of memory, not a replacement — so there's nothing to
clash with."
