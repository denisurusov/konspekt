**User:** Explain the first one

**Claude:** "[…] **What it costs.** You can't turn an edge reference into a file path by string manipulation — `noteworthy:nw-timestamp-source-time` doesn't lead to `noteworthy/nw-timestamp-source-time.md`. Any consumer has to open every file and build an id→path index first. You also lose `ls noteworthy/` as a way to read the inventory without parsing anything […]. **Three ways out, very different prices.** Rename the 48 files to match their ids: mechanical, no content edits, no edge changes, because ids don't move. Or strip the prefixes from the ids: 48 front-matter edits plus every `type:id` reference in `edges.md` and in entity bodies — far larger and the only option that can break references. Or amend the rule […]. **One correction to what I wrote into the atom.** I claimed renaming would be a breaking layout change requiring a v2 bump. I think that's wrong. The spec already says filenames equal ids, so renaming brings the tree into line with the existing v1 layout — that's conformance repair, not a layout change. Amending the rule is the option that actually redefines layout. The atom has it backwards […]."
