KONSPEKT

A portable, human-readable record of a project’s evolving state across generative-AI conversations and platforms.

Conversation → Claude proposes — verb → Human accepts → Commit to GitHub
Data model — graph01

A typed knowledge graph. Node kinds and entity types wired through a single typed edge table. Inventories are queries over edges, never stored lists.

goalinvestigationexperimenttopictasknote
Verbs — commands02

Human-authority verbs. Claude proposes, the human accepts; self-acceptance is never permitted. An accepted verb lands as review: accepted.

pinvalidaterefuteresolveabandonlift
Provenance — citations03

Every atom cites its source. Verbatim excerpts from both sides of the exchange, content-addressed by git blob SHA, tied to the conversation. A paraphrase would be an atom in disguise.

contentHashsourceRefconversationIdtimestamp
GitHub — backend04

The store is the repo. Atomic multi-file commits, blob SHA as content hash, human-readable files. Distribution is a regenerable projection of root, never a tracking copy.
